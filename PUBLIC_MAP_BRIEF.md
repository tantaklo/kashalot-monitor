## ⚠️ ГЛАВНОЕ ПРАВИЛО БЕЗОПАСНОСТИ (не нарушать)

Внутренний дашборд (`dashboard.html`) обращается к базе через Grafana-прокси, и **токен `X-Token` лежит прямо в клиентском JS**. Для внутреннего инструмента под паролем (staticrypt) это допустимо.

**Для публичного продукта это НЕДОПУСТИМО.** Если выложить карту наружу с тем же подходом — любой откроет DevTools, достанет токен и получит доступ ко всей базе (заказы, выручка, все устройства, персональные данные).

Поэтому:
- Клиентская страница **НЕ** содержит никаких токенов, паролей, ключей.
- Все запросы к базе идут через **новый публичный эндпоинт Worker**, который сам ходит в базу (секрет остаётся на сервере).
- Публичный эндпоинт отдаёт **только минимум**: координаты + примерный заряд/доступность. Никаких внутренних ID, госномеров, выручки, company_id.

## Технические факты (уже выяснены, можно опираться)

### Где живёт инфраструктура
- Репо: `tantaklo/kashalot-monitor` (`Projects/monitor/`)
- Cloudflare Worker: `kashalot-proxy.vegano-argentino-ru.workers.dev`, файл `worker.js`
- Deploy worker: `cd Projects/monitor && npx wrangler deploy` (git push НЕ деплоит worker!)
- Deploy страниц: `git push` → GitHub Actions → GitHub Pages
- KV Namespace `IGN_CACHE` (id `517109d7aaa940729eae828d1a452aae`) — для edge-кеша

### Координаты и доступность в БД
Таблица `cars` (~3000 трекеров, все Кашалот). Нужные поля:
| Поле | Описание |
|---|---|
| id | ID устройства (КШ-{id}) — **наружу не отдавать** |
| gosnomer | Госномер — **наружу не отдавать** |
| lat, lon | GPS-координаты (НЕ lng!). Бывают NULL или 0 — фильтровать |
| fuel | Заряд %, 0 = выключен/на хранении |
| online | 1 = онлайн сейчас |
| cur_order_id | NULL = свободен; не NULL = сейчас в аренде |
| hidden | 1 = скрыт, не показывать |

### SQL «свободные к аренде» (проверено, работает)
```sql
SELECT id, gosnomer, fuel, lat, lon
FROM cars
WHERE online = 1
  AND cur_order_id IS NULL
  AND lat IS NOT NULL AND lat != 0
  AND lon IS NOT NULL AND lon != 0
  AND (hidden IS NULL OR hidden = 0)
  AND fuel > 5
ORDER BY fuel DESC
LIMIT 600
```
Для публичной версии из SELECT убрать `id` и `gosnomer` — отдавать только `lat, lon, fuel` (или округлённый «уровень заряда»: высокий/средний).

### Рекомендуемая архитектура публичного канала
```
Клиент (find.html, без токенов)
   │  fetch('/public/map?lat=..&lon=..')   ← геолокация клиента
   ▼
Worker эндпоинт /public/map
   │  1. читает KV-кеш public_map (TTL 60–120 сек)
   │  2. если протух — запрос в Grafana секретным токеном (на сервере)
   │  3. фильтрует поля → только lat, lon, заряд
   │  4. (если выбран радиус) отдаёт только точки рядом с клиентом
   ▼
JSON минимум → рисуем маркеры
```
- Кеш в KV: 10 000 посетителей = несколько запросов в базу. Нагрузки почти нет.
- Карта: Leaflet + CartoDB (бесплатно, без API-ключа) — как во внутренней карте.
